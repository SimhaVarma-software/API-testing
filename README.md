
# Task: Setting up and Pushing Changes to GitHub with AI Review

This document outlines the steps required to complete the task and create a Pull Request (PR) with an AI review.

---

## Steps to Complete the Task

### 1. Create a New Branch Locally
- Create a new branch with an **appropriate and descriptive name** based on the task.
  - Example: `feature/add-docs-bruno`.

### 2. Create a Folder
- In your local repository, create a folder named **`docs.bruno`**.

### 3. Add Request Files
- Add the request files (e.g., `/ping`) generated by Bruno into the `docs.bruno` folder.
  - These request files can typically be found in the location where you initially created your collection.

### 4. Commit Your Changes
- Stage the changes:
  ```
  git add docs.bruno
  ```
- Commit the changes with a meaningful commit message:
  ```
  git commit -m "Add request files to docs.bruno"
  ```

### 5. Push the Branch to GitHub
- Push the newly created branch to the remote repository:
  ```
  git push origin feature/add-docs-bruno
  ```

### 6. Create a Pull Request
- On GitHub, create a **Pull Request (PR)** from your branch to the `main` branch.
  - Ensure the `main` branch is named `main` (not `master`).

### 7. Request an AI Review
- Add a comment in your PR to request an AI review:
  ```
  @CodiumAI-Agent /review
  ```
  - This comment will trigger an AI review for your PR.

### 8. Record a Video
- Record a video where you:
  - Explain the steps you took to complete the task.
  - Showcase the changes you’ve made in the branch and PR.

---

## ⚡ Important Notes
1. Use an **appropriate naming convention** for your branch.
2. **Merge the PR** into the `main` branch after addressing all feedback.
3. Double-check that the `main` branch is named `main` (not `master`).
